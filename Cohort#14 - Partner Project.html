Working as a team
1- Team leader will set up the project repository.
Team lead will create a new repository (find a project name).
Team lead will add collaborators
Go to Settings -> Collaborators
You gave them push access to the repository. (and other rights)
Collaborators will have to accept your invitation.
Team lead will make the initial commit to the project after creating the html,css and js files.
2- Other team members will add the project locally via a clone.
Open the terminal
Change the current working directory to the location where you want the cloned directory.
Type git clone, and then paste the URL you copied earlier.
git clone git@github.com:mrselifbalci/quote-generator.git
Plain Text
If you have an error regarding SSH, reach out to your instructor for support.
Open the project in VS code.
2- Working on a new branch.
When you clone the repository, you‚Äôre by default on a branch, main.
RULE NUMBER 1: Never commit directly to master/main. Use feature branches.
Branches allow us to work on different parts of a project at the same time.
üí°
Think of a branch as a Save as function that you use with PowerPoint. You create a copy of original and each change to make to the copy does not affect the original.
One feature === one branch
For example, you want to customise the navbar.
git checkout -b customise-navbar // you created a new local branch called customise-navbar
Plain Text
Listing local branches
git branch
Plain Text
3- You finished working on the feature
Now you made some changes to your code, for example, you changed the background of the navigation to be green. And you want to merge commits back in main.
git add . git commit -m "change background color of the navbar" git push origin customise-navbar
Plain Text
Next steps:
Create a Pull Request (1)
As soon as you push a new branch, a pull request button appears on your GitHub repository page.
Just click on Compare & pull request button, review the diff and add clear title and description.
Creating a Pull Request (2)
Take some time to write a proper title and description
Explain what you did on the branch (new component, change a color, etc‚Ä¶)
You can poke people with @‚Ä¶, to get their feedback
Merging the changes
The team leader will review the Pull Request and merge the changes.
3. Getting main up to date
When a Pull Request is merged, a new commit is created on main. You need to fetch it on your local repository.
Be very careful First, you need a CLEAN git status.
git status /# On branch customise-navbar/ /# Your branch is up-to-date with ‚Äòorigin/customise-navbar‚Äô./ /# nothing to commit, working tree clean/
Plain Text
Get the latest master
/# Remember! You must have a **clean** git status/ git checkout main git pull origin main
Plain Text
4. In case of conflict
You can also solve conflicts on Github. Click on Resolve conflicts.
Keep only the code you want to keep, and remove the special characters that highlighted the conflict.
When it‚Äôs done, click on Mark as resolved and Commit merge.
